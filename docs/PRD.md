# 产品需求文档 (PRD)

> 项目名称：CAACI Community Marketplace Bazaar
> 版本：v0.1 (初稿)
> 最后更新：2025-04-13
> 状态：规划中

---

## 1. 产品概述

### 1.1 产品愿景

打造面向北美华人社区的同城二手交易与社区互动平台，结合闲鱼（二手交易）和小红书（内容社区 + 获客逻辑）的核心体验，服务于华协 (CAACI) 及其辐射的本地华人群体。

### 1.2 目标用户

| 用户类型 | 描述 |
|---------|------|
| **普通用户 (C端)** | 本地华人居民，有二手交易、社区交流需求 |
| **华协会员** | CAACI 订阅会员，享受额外福利 |
| **本地商户 (B端)** | 希望触达华人社区的本地商家 |

### 1.3 核心策略

- **初期**：纯信息发布平台，不涉及平台内交易，降低合规和支付复杂度
- **中期**：逐步加强信息审核，引入商户合作体系
- **长期**：成为本地华人生活服务一站式平台

## 2. 功能模块

### 2.1 底部导航 Tab 结构 (4 个)

```
┌─────────────────────────────────────────────┐
│                  内容区域                      │
├──────────┬──────────┬──────────┬─────────────┤
│  🏷 二手  │  🏠 社区  │  💬 私信  │  👤 我的    │
└──────────┴──────────┴──────────┴─────────────┘
```

---

### Tab 1: 二手交易

**核心功能**：发布、浏览、搜索二手物品

- 发布商品（图片 + 描述 + 价格 + 位置 + 分类）
- 瀑布流 / 列表切换浏览（参考小红书布局）
- 搜索 + 多维度 Filter：
  - 分类（家具、电子、服饰、书籍、转租...）
  - 价格区间
  - 距离 / 区域
  - 发布时间
  - 物品成色
- 收藏 / 感兴趣标记
- 已出 / 未出状态管理

**分类包含**：
- 二手物品交易
- 房屋转租
- 求购信息

---

### Tab 2: 社区论坛

**核心功能**：信息交流 + 官方便民信息

**顶部区域 — 官方信息频道**：
- 官方审核的便民信息（招聘、服务推荐等）
- 本地优质商家每周推荐
- 当地新闻自动聚合（Agent 爬取）
- 点击可展开/跳转详情

**下方区域 — 用户论坛**：
- 用户自由发帖
- 帖子评论/回复
- 分类/话题标签
- 内容类型：求助、分享、讨论、活动通知等

---

### Tab 3: 私信

**核心功能**：用户间一对一即时通讯

- 会话列表
- 实时聊天（文字 + 图片）
- 从二手帖 / 论坛帖直接发起私信
- 未读消息提醒

---

### Tab 4: 个人中心（我的）

**核心功能**：用户管理 + 会员体系

- 个人资料编辑
- 我发布的（二手 + 论坛帖子）
- 我的收藏
- **华协会员卡**：
  - 会员等级展示
  - 订阅状态 & 续费情况
  - 可用 coupon 列表
  - 二维码（商家扫码核销）
  - Apple Wallet 集成（理想状态）
- 设置 & 通知管理

## 3. 会员体系（面向 C 端用户）

| 等级 | 费用 | 权益 |
|------|------|------|
| **免费用户** | ¥0 | 注册、发布二手、发帖、浏览、私信 |
| **基础会员** | 月费 TBD | 餐厅 coupon、商家优惠、每月福利限额 |
| **高级会员** | 月费 TBD | 更大折扣、活动优先入座/好位置、商家新品优先体验 |
| **至尊会员** | 年费 TBD | 定制周边（扑克牌等）、年度礼品、全部高级权益 |

### 关键设计原则

- 免费用户**可以**发布二手物品和论坛帖子（鼓励用户增长）
- 会员权益聚焦**消费者福利**（coupon、折扣、优先体验）
- **曝光/推广**属于商户体系，与会员体系分离

---

## 4. 商户合作体系（面向 B 端）

> ⚠️ 与会员体系严格分离，针对不同用户群

### 合作模式

- **官方推荐**：每周本地商家推荐（网站/小红书/私域群同步）
- **付费投流**：精细化广告投放，独立收费标准
- **活动合作**：华协活动中商家提供产品展示/试用
- **成本价福利**：商家以成本价供货 → 作为高级会员福利
- **个人急卖/推广**：按商户模式对接，不归入会员权益

## 6. 安全 & 审核

### 6.1 用户注册与验证

- 必填：手机号、邮箱
- 基本实名：确保可追溯性
- 华协会员可关联会员账户

### 6.2 信息真实性

- **初期策略**：平台免责声明，用户自行判断
  > "本平台仅提供信息发布服务，不对信息真实性负责，请用户自行甄别"
- **后续优化**：逐步引入信息审核机制

### 6.3 内容审核

- 初期方案待定：人工审核 vs AI 自动化
- 需覆盖：违规言论、虚假信息、违禁品等

### 6.4 网络安全

- 人机验证 (CAPTCHA)
- 网络安全套餐（WAF、DDoS 防护等）
- **⚠️ 关键原则**：用户个人信息必须存储在服务器端，AI 工具（如 OpenCloud）不可直接访问用户数据

## 7. 技术架构概要

### 7.1 AI 辅助开发架构

```
OpenCloud (主控, OpenRouter Key)
  └── OpenCoder (作为 skill/工具)
        └── Claude Max Subscription
              └── 执行项目开发任务
```

### 7.2 数据安全隔离

```
┌──────────────┐     ┌──────────────┐
│  AI 工具层    │  ✗  │  用户数据层   │
│  (OpenCloud)  │────→│  (服务器端)   │
│              │ 禁止 │  个人信息     │
│              │ 直接 │  手机/邮箱    │
│              │ 访问 │  交易记录     │
└──────────────┘     └──────────────┘
```
