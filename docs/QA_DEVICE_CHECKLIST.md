## 1. 报价系统(structured offers) — 四态全跑

买家在商品详情页或聊天里发起报价,卖家在聊天里看到报价卡操作。

| # | 场景 | 操作 | 预期 |
|---|---|---|---|
| 1.1 | 发起报价 | 买家在可议商品上发报价 $X | 聊天出现报价卡(金额 / 等待回复 / 24h 有效期),卖家侧实时收到 |
| 1.2 | 卖家接受 | 卖家点「接受」 | 双方聊天出现成交系统消息「🎉 双方达成 $X」,状态变 accepted |
| 1.3 | 卖家拒绝 | 卖家点「拒绝」 | 报价卡变 declined,买家侧实时更新 |
| 1.4 | 还价 | 卖家点「还价 $Y」 | 生成新报价卡(链到原报价),买家可再接受/拒绝 |
| 1.5 | 24h 过期 | 放置过期(或改系统时间) | 报价卡变 expired,不能再操作 |
| 1.6 | 快捷出价 | 买家用挂牌价 70/80/90% 快捷按钮 | 金额自动填入,能正常发出 |
| 1.7 | 非参与方 | 第三个账号尝试读这条会话的报价 | 读不到(RLS 挡住),不报错白屏 |

## 1b. 见面约定(meetup,migration 052 — 先确认线上库已跑 052 再测)

聊天里点蓝色「📍 约见面」发起,对方在聊天里看到约见卡操作。

| # | 场景 | 操作 | 预期 |
|---|---|---|---|
| 1b.1 | 发起约见 | A 选安全点 + 日期 + 时间发出 | 聊天出现约见卡(地点/时间/待确认/24h),B 侧实时收到 |
| 1b.2 | 确认 | B 点「确认」 | 双方出现「🤝 约定 …」系统行,状态 accepted |
| 1b.3 | 婉拒 | B 点「婉拒」 | 卡变 declined,A 侧实时更新 |
| 1b.4 | 改约 | B 点「改约」选新地点/时间 | 生成新约见卡(链到原约),A 可再确认/婉拒 |
| 1b.5 | 双方都能发 | 卖家(商品主)也点「约见面」 | 卖家也能发起(报价只买家能发,约见两边都能) |
| 1b.6 | 地点预填 | 商品自取点是已知安全点 | 打开约见面板时该点已高亮选中 |
| 1b.7 | 24h 过期 / 非参与方 | 同报价 1.5 / 1.7 | 过期不能操作;第三方读不到不白屏 |
| 1b.8 | 通知 | 收到约见/确认 | 通知中心出现 📍 见面约定条目 |

## 3. 桌面两栏消息(≥768px,iPad 横屏 / Mac)

逻辑是从单页聊天逐字节搬到 ChatThread 组件的,登录态实时流没自动测过,重点验这块。

| # | 场景 | 操作 | 预期 |
|---|---|---|---|
| 3.1 | 选会话 | 桌面宽度点左侧会话 | 右栏内嵌打开对话,不跳页 |
| 3.2 | 实时收消息 | 对方发消息 | 右栏实时出现,不用刷新 |
| 3.3 | 切会话 | 点另一个会话 | 右栏切换,**不出现上一条会话的消息串台 / 气泡重复** |
| 3.4 | 报价弹层 | 在右栏内发报价 | 弹层只盖住对话面板,不盖全屏侧栏 |
| 3.5 | 拖窗变窄 | 桌面拖到 <768px | 回到手机式列表,再点会话跳页,不残留双重订阅 |
| 3.6 | 手机端 | 真手机点会话 | 照旧跳到独立聊天页,行为不变 |

## 5. 卖家「回复率」指标(已上线 — migration 053)

详情页卖家卡的「X% 回复率」**已实现**:migration 053 用 trigger 把 `response_rate` / `response_sample` 反范式到 `profiles`(RLS 挡客户端读别人会话,所以反范式 + 列级 SELECT grant)。**样本 < 5 个会话时隐藏**,避免「1 个会话 100%」误导。

| # | 场景 | 操作 | 预期 |
|---|---|---|---|
| 5.1 | 有样本的卖家 | 打开一个被 ≥5 个不同买家发起过会话、且卖家有回过的卖家详情页 | 卖家卡社会证明行出现「X% 回复率」 |
| 5.2 | 样本不足 | 打开一个几乎没人私信过的卖家详情页 | 不显示回复率(只显示已售/评分) |

## 6. iOS 软键盘遮挡(QA4 B1/B2 — 真机必验,桌面/模拟器看不出)

照 plaza 成熟方案(`useKeyboardHeight` + `translateY(-kb.height)`),把这几个底部输入区在键盘弹起时整体上移到键盘之上;受影响的 textarea/input 已关掉 `adjust-position`(改由 transform 单独抬,避免 mp 上双重抬升)。**只有真 iPhone 能验**:重点看输入框/发送键是否被键盘盖住、上移是否跟手不抖、收键盘是否平滑回落。

| # | 场景 | 操作 | 预期 |
|---|---|---|---|
| 6.1 | 聊天输入 | 真机进对话,点底部文本框 | 输入栏 + 发送键整体浮到键盘正上方,不被盖;打字可见 |
| 6.2 | 聊天联想行 | 输入触发 emoji 联想 chips 时弹键盘 | 联想行跟输入栏一起上移,不错位 |
| 6.3 | 报价弹层 | 对话里发起/还价报价,点价格框 | 报价 sheet 抬到键盘上方,价格/备注框可见可输 |
| 6.4 | 见面弹层 | 发起/改约见面,点地点或备注框 | 见面 sheet 抬到键盘上方,地点/备注可输;日期/时间 picker 不被压 |
| 6.5 | 帖子评论 | 帖子详情点底部评论框 | 评论栏浮到键盘上方,发送键可点 |
| 6.6 | 收键盘 | 上述任意场景点空白/发送后 | 输入区平滑回落原位,不残留偏移、不抖 |
| 6.7 | 桌面回归 | 桌面宽度(有硬件键盘)进聊天/帖子 | 无任何上移(kb.height=0),布局照旧不变 |
| 6.8 | plaza 回归 | plaza 发动态 composer 点输入 | 原有键盘避让仍正常(本轮未改 plaza,验证无回归) |
